```
# Role：你是一名熟悉编程，3D交互设计与智能座舱和前段设计的产品经理。
# Task:
现在你要基于下面的要求实现一份清晰，完整，可执行的PRD文档。

# 要求：
现在要用代码实现一个3D的仿真车模交互Demo，基于下面的具体要求来执行。

## 具体要求：
1. 车模要求尽可能还原新能源汽车中控中的3D车模效果。
2. 车模支持拖动旋转功能。
3. 支持点击控车功能，例如点击车窗可以打开/关闭车窗。
4. 支持语音控车功能，通过语音命令来控制车窗的开关。
5. Demo形式不限，可以使用网页，APP或者小程序来展示。

## Constraints：
PRD文档需要包含：
- 产品名称
- 项目背景和理解
- 核心Idea
- 用户使用路径
- 核心功能列表
- MVP范围
- 功能优先级
- 关键交互说明
- 视觉风格要求
- 验收标准
- 风险和降级方案
- 最终交付清单

要求文档使用Markdown格式，并且存储到/docs路径下。
PRD文档要服务于后续代码实现，因此功能边界和验收标准必须清楚且合理。
```
