## 3. 实现了哪些关键逻辑

- 四窗初始状态统一为 `closed`；所有输入共用 `VehicleCommand` 和 `executeCommand`。
- 每个过渡记录 `previous` 与 `target`：开始时公开状态为 `transitioning`，成功时收敛目标，失败时回退上一个稳定状态。
- 同窗过渡期间拒绝新命令，异窗仍可独立开始；全窗命令跳过锁定和已满足车窗，只启动其余目标。
- 执行结果使用 `accepted / partial / noop / blocked`，并返回只读的 `started`、`skipped`、`alreadySatisfied` 列表供反馈层使用。
- `WINDOW_IDS` 使用只读稳定顺序；store 不保存 Three.js 原生对象。

### Sub-agent 使用情况

- 委派内容：Sub-agent 仅在 `src/domain/vehicle/**` 和领域出口内实现类型、store 与测试，并运行领域验证。
- 主代理审查：检查 PRD 契约、过渡收敛和结果分类；要求补充只读公共契约及全窗完整执行测试，随后独立运行领域测试和全量质量门禁。
