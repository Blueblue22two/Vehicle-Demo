- 目标：接入授权清晰、适合 Web 渲染且具备四个独立可动车窗节点的新能源概念车 GLB。

## 3. 实现了哪些关键逻辑

- 原始资产为 Khronos glTF Sample Assets 的 CarConcept，由 Eric Chadwick / Darmstadt Graphics Group GmbH 创建，采用 CC BY 4.0；版权字段继续内嵌在 GLB。
- 将 `BodyDoorLWindow`、`BodyDoorRWindow` 映射为左前和右前车窗固定节点。
- 将原合并后侧窗的 1,376 个三角形按本地 `POSITION.x` 分为左右各 688 个；不存在跨中线三角形，两个后窗使用独立索引访问器、Mesh 和节点。
- 适配器保持原几何属性、Glass transmission 材质、内嵌纹理、节点变换和 glTF 扩展，并记录 `asset.extras.neocabinAdaptation`。
- 校验器检查文件大小、GLB 结构、chunk 对齐、buffer/accessor 边界、所有 Mesh 引用、索引范围、四窗唯一性和独立性、玻璃透明材质、分侧三角形与整体有限包围盒。

### Sub-agent 使用情况

- 委派内容：Sub-agent 负责限定范围内的 GLB 二进制适配脚本、模型校验器、最终资产和授权说明。
- 主代理审查：逐段复核 chunk 重建、索引拆分和许可说明；从原始文件独立再生成并比对哈希，使用 Three.js GLTFLoader 解析最终场景，重跑模型门禁、构建和全量回归。

## 6. 是否有遗留问题

- 已知问题：Node 环境不提供浏览器图像解码器，GLTFLoader 命令行解析时会提示内嵌纹理无法解码；场景和四窗几何仍成功解析，浏览器加载不受此 Node 限制。
- 已接受风险：原模型含 Khronos 标志，演示与再分发必须保留归属和非背书声明，并遵守 Khronos 商标规则。
- 阻塞项：无。
