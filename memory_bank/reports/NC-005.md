## 3. 实现了哪些关键逻辑

- **场景容器**：Canvas 使用 ACESFilmicToneMapping + SRGBColorSpace，dpr 上限 1.5，透视相机 fov=40、near=0.5、far=50，默认位置 (5, 2.5, 5) 提供左前 45° 构图。
- **灯光系统**：ambientLight(0.4, 冷白) 作为环境填充；主 directionalLight(2.5) 从 (8,12,6) 投射 1024×1024 阴影；补光 directionalLight(0.8) 从 (-4,3,-4) 减少暗部过黑。
- **地面与阴影**：透明 shadowMaterial plane 接收阴影 + Drei ContactShadows 提供柔和接触阴影。
- **模型加载与布局**：useGLTF 带缓存加载；遍历所有 Mesh 计算世界包围盒；等比缩放使最大维度为 3.5 单位；X 和 Z 方向居中，Y 方向将包围盒底部对齐 y=0（保留车轮落地）；使用外层 Group 包装避免修改缓存的原始 scene。
- **加载状态**：Suspense 捕获 useGLTF 的异步挂起，显示加载占位文案；加载完成前不显示模型。
- **错误处理**：ErrorBoundary 捕获任意后代渲染错误；显示错误信息、原因和"重新加载"按钮；错误状态下不渲染任何车控交互（Canvas 不可见）；retry 通过递增外层 key 强制完全重新挂载 ErrorBoundary + Suspense + Canvas 子树。
- **测试策略**：mock @react-three/fiber（Canvas → div）和 @react-three/drei（useGLTF 可控、ContactShadows → null）；覆盖正常渲染、Suspense 挂起、同步错误捕获、异步重试触发、卸载清理。

### Sub-agent 使用情况

- 委派内容：本任务由主代理直接实现，未委派 Sub-agent。原因：五个文件边界清晰且互相依赖（VehicleScene 依赖 VehicleModel、ErrorBoundary、LoadingFallback），拆分委派会引入跨文件协调成本。
- 主代理审查：自检所有组件类型、测试覆盖、包围盒计算和错误恢复路径。
