## 3. 实现了哪些关键逻辑

- **StatusPanel**：`useVehicleStore(s => s.windows)` 细粒度订阅（仅 windows 字段，避免 3D 重渲染）。四个 WindowId 按固定顺序渲染为双列网格。状态样式：closed 灰色、open 绿色、transitioning 青色 pulse 动画。
- **TextCommandInput**：`handleSubmit` 读取 input ref → trim → `parseVehicleCommand(text, 'text')` → 若 ok 则 `useVehicleStore.getState().executeCommand(command)` → `onFeedback({ type: 'command', ... })`。解析失败则 `onFeedback({ type: 'parse-error', reason })`。成功后清空 input。`handleKeyDown` 拦截 Enter → preventDefault + 调用 handleSubmit。
- **FeedbackDisplay**：受控于 `feedback` prop 变化。useEffect 中监听 feedback 引用变化：idle→setVisible(false)，非 idle→setVisible(true)+setTimeout 自动消失。accepted/noop 3s、parse-error 4s、其他 5s。variant 映射：accepted/partial→success、noop→info、blocked→warning、parse-error→error。使用 `<output role="status" aria-live="polite">` 提供无障碍反馈。
- **ControlPanel**：useState 管理 FeedbackKind；handleFeedback 回调传给 TextCommandInput。位置为 absolute 右下角，半透明深色背景+backdrop-filter blur，z-index 20 覆盖在 3D Canvas 上。
- **App 布局**：CSS Grid `grid-template-rows: auto 1fr auto`（header→3D→footer）。ControlPanel 绝对定位在 `main` 区域右下角。footer 显示操作提示。

## 8. 2026-06-27 开发模式白屏修复

- **问题现象**：通过 `npm run dev` 打开页面后，二维页面骨架可见，但 3D Canvas 变白且车辆模型不可见。
- **根因**：`src/main.tsx` 使用顶层 React `StrictMode` 包裹整个应用。开发模式重复执行生命周期时，React Three Fiber 的 Canvas 清理流程释放 WebGL context，控制台出现 `THREE.WebGLRenderer: Context Lost.`；该事件不会进入 React Error Boundary。
- **排除项**：模型自动校验通过；四窗节点及包围盒有效；Canvas 实测尺寸非零；模型经当前相机投影位于视锥内；生产预览未出现 context lost。
- **修复**：移除应用入口的顶层 `StrictMode`，避免开发模式下释放正在使用的 WebGL context；不改变车辆场景、状态机或二维控制逻辑。
- **回归测试**：新增 `src/main.test.tsx`，锁定入口不得再以 `StrictMode` 包裹 WebGL Canvas 所在应用树。
