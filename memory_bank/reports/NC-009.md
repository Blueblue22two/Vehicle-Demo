## 3. 实现了哪些关键逻辑

- **SpeechAdapter 接口**：`supported` / `state` / `start()` / `stop()` / `subscribe(listener)`。业务代码只依赖此接口，适配器可替换（CI mock、未来云端 ASR）。
- **BrowserSpeechAdapter**：构造函数检测 `SpeechRecognition` 和 `webkitSpeechRecognition`——无 API 时 `supported=false` 且 `state='unsupported'`。`start()` 流程：停止前次→创建实例（lang='zh-CN', continuous=false, interimResults=false）→`setState('permission')`→调用 `start()`→`onstart` 时 `setState('listening')` + 启动 10s timeout→`onresult` 时区分 isFinal（interim 忽略）→ emit result→`onerror` 时映射错误码→`onend` 时清理 timeout。`stop()` 中止识别并 null 所有 handler。`subscribe()` 返回 unsubscribe。
- **错误码映射**：`not-allowed`/`service-not-allowed`→`denied`；`no-speech`→`no-speech`；`aborted`→`aborted`；`network`→`network`；`audio-capture`/`bad-grammar`/`language-not-supported`/其他→`not-supported`。
- **VoiceControl 组件**：`useState(() => new BrowserSpeechAdapter())` 懒初始化保证单例。`useEffect` 中 subscribe 适配器事件：statechange→更新 UI 状态；result→`parseVehicleCommand(text, 'voice')`→成功则 `executeCommand` + `onFeedback({ type:'command' })`，失败则 `onFeedback({ type:'parse-error' })`；error→设置 `errorType`（`denied` 禁用按钮，其余可重试）；end→若在 listening/success 中则回 idle。`handleClick` 中 listening/permission 时 stop，其他时 start。Unmount 时清理订阅并 stop。
- **状态 UI 映射**：idle→"点击麦克风开始语音控制"；permission→"等待麦克风授权…"；listening→"正在聆听…"+button--active+pulse 动画；success→"识别成功"（瞬时，结果通过 FeedbackDisplay 展示）；error+denied→"麦克风权限被拒绝…"+disabled；error+其他→对应中文提示+可重试；unsupported→"浏览器不支持…"+disabled。
- **与现有系统集成**：语音识别的中文文本进入与文本输入**完全相同**的 `parseVehicleCommand(text, 'voice')` 解析和执行链路。VoiceControl 置于 ControlPanel 中 TextCommandInput 上方，command 反馈复用 FeedbackDisplay。
- **降级保证**：不支持 API 或拒绝权限时麦克风按钮禁用，文本输入和点击控车完全不受影响。任何语音错误均不改变车辆状态。

### Sub-agent 使用情况

- 委派内容：本任务由主代理直接实现，未委派 Sub-agent。原因：语音适配器接口清晰、组件边界明确、与现有系统耦合点有限，主代理直接实现更高效。
- 主代理审查：自检 adapter 状态机完整性、VoiceControl 懒初始化避免 ref-in-render lint 错误、错误码映射覆盖所有 SpeechRecognitionError 类型、parse→execute 链路与文本输入一致、降级路径不影响点击和文本入口。

## 6. 是否有遗留问题

- 已知问题：无阻断性问题。ESLint 有 4 个 warnings（全部预存于 WindowInteraction.tsx，与 NC-009 无关）。VoiceControl 中 `useState(() => new BrowserSpeechAdapter())` 懒初始化模式在 React strict mode 下会 double-invoke 构造函数，但 adapter 实例最终取第二次结果，无副作用（适配器构造函数仅检测 API 存在性，不发起网络请求）。
- 已接受风险：
  - **真实语音识别验收需人工执行**：Web Speech API 依赖浏览器权限弹窗、麦克风硬件和网络语音服务，无法在 CI 环境自动化。NC-011（自动化测试与回归）将建立人工验收步骤和记录模板。当前 mock 测试覆盖全部状态转换和降级路径。
  - **浏览器兼容性**：语音功能仅正式支持最新版桌面 Chrome（localhost 或 HTTPS），Safari/Firefox 中 SpeechRecognition 不可用，voice 组件自动降级为 unsupported 状态。
- 阻塞项：无。
