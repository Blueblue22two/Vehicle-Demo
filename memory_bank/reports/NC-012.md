## 3. 实现了哪些关键逻辑

- **真实加载进度**（`SceneLoadingFallback`）：
  - 使用 Drei `useProgress()` hook 获取 `{ progress: 0–100, active: boolean }`
  - active 时显示 "正在加载 3D 车辆… 45%"，非 active 时显示静态文案
  - 进度条 `<div role="progressbar" aria-valuenow={percent}>` 配合青色填充条 `width: ${percent}%` + CSS `transition: width 0.3s ease`
  - 失败时切换到错误状态（由 SceneErrorBoundary 处理）

- **视角重置**（`CameraController` + `ResetViewButton`）：
  - `VehicleScene` 维护 `resetTrigger` state（每次点击 +1）
  - `CameraController` 接收 `resetTrigger` 变化，在 `useFrame` 中检测触发
  - 动画：记录起始 `camera.position` 和 `controls.target` → ease-in-out cubic 插值 → 终点 `[5, 2.5, 5]` / `[0, 0, 0]`
  - 动画期间 `controls.enabled = false` 防止用户旋转冲突
  - 动画速度由 `RESET_LERP_SPEED = 4` 控制，约 0.25s 完成（`delta * 4` 累积到 1.0）
  - `ResetViewButton` 置于场景容器左上角：半透明深色背景 + backdrop-filter blur + SVG 旋转图标

- **座舱视觉增强**（`SceneLights`）：
  - 环境光：`intensity=0.35, color=#8899cc`（降低以增加对比度）
  - 主光：`intensity=3.0, color=#fff8f0`（暖白，upper-right-front），阴影 2048×2048 + normalBias
  - 背光：`intensity=1.2, color=#6688cc`（冷蓝 rear-left，勾勒深色车身轮廓）
  - 补光：`intensity=0.6, color=#aabbdd`（below-front 柔和填充）

- **接触阴影增强**：`shadowMaterial opacity=0.3`、`ContactShadows opacity=0.45 blur=2.5`

- **悬停增强**：hover emissive `#3388aa`（更亮的青蓝），intensity `0.55`（更显著的发光）

## 6. 是否有遗留问题

- 已知问题：无阻断性问题。ESLint 有 4 个 warnings（全部预存于 WindowInteraction.tsx）。`CameraController` 中 `controlsRef` 使用 `any` 类型（Drei 的 OrbitControls ref 类型复杂，已加 eslint-disable 注释）。
- 已接受风险：
  - **精细描边未实现**：精细车窗描边需要后处理管线（EffectComposer + OutlinePass），显著增加包体和 GPU 压力。当前 emissive 提亮方案已满足悬停反馈需求。如需描边，可在后续版本中作为可选增强。

- 阻塞项：无。
