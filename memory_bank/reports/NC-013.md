## 1. 本阶段完成的任务

- 目标：达到首次可交互、帧率和三种桌面视口要求，并验证语音能力之外的浏览器降级。
- 完成情况：
  1. **性能自适应**：将 `performanceConfig.ts`（基于 `hardwareConcurrency` + `deviceMemory` 的 three-tier 检测）接入 VehicleScene。Canvas dpr、shadow map 分辨率、阴影开关随 tier 动态调整。降级顺序：pixel ratio（high/medium 1.5→low 1.0）→shadow map（2048→1024→512）→shadows disabled（low tier）。
  2. **响应式 CSS**：新增 `@media (max-height: 800px)` breakpoint 针对 1280×720 视口压缩控制面板（padding、gap、font-size、button size 缩小），`@media (min-width: 1800px)` 为 1920×1080+ 视口提供宽松间距。
  3. **React 渲染优化**：`StatusPanel` 包装 `memo()` 避免因非 windows 状态变更导致的不必要重渲染。`useVehicleStore` 选择性订阅、`useMemo` 缓存计算、`ref` 驱动动画已在前序任务中完成。
  4. **浏览器降级**：语音不可用时 VoiceControl 自动 unsupported+disabled（NC-009 实现）。点击和文本入口始终可用。E2E 测试验证。
- 范围偏差：无。`createFpsSampler` 已在 `performanceConfig.ts` 中定义并测试，作为 dev 工具保留。

## 3. 实现了哪些关键逻辑

- **性能自适应层级**（`performanceConfig.ts` 已于 NC-013 前完成，本次集成到 VehicleScene）：
  - `getPerformanceConfig()` 在 Mount 时通过 `useMemo` 调用一次，缓存结果。
  - **High**（≥8 cores 或 ≥4GB）：`maxDpr=1.5`, `shadowMapSize=2048`, `shadowsEnabled=true`
  - **Medium**（≥4 cores，default）：`maxDpr=1.5`, `shadowMapSize=1024`, `shadowsEnabled=true`
  - **Low**（<4 cores）：`maxDpr=1.0`, `shadowMapSize=512`, `shadowsEnabled=false`
  - Canvas `dpr` 从硬编码 `[1, 1.5]` 改为 `[1, perf.maxDpr]`
  - SceneLights 动态接收 `shadowMapSize`/`shadowsEnabled`
  - SceneGround 在 `!shadowsEnabled` 时跳过 shadowMaterial plane

- **响应式 CSS**：
  - `@media (max-height: 800px)`：控制面板 bottom 2.5→0.75rem、padding 1→0.6rem、gap 0.75→0.4rem、min-width 260→220px、max-width 300→260px；状态面板/标签字体缩小；输入框/按钮字体 0.85→0.75rem；语音按钮 44→36px
  - `@media (min-width: 1800px)`：控制面板 min-width 260→280px、padding 1→1.25rem、gap 0.75→1rem

- **React 优化**：
  - `StatusPanel` 使用 `memo()`：无 props，仅依赖 store subscription，`windows` 不变时不重新渲染
  - `VehicleModel.useMemo` 缓存 position/scale 计算（已有）
  - `WindowInteraction` 使用 ref 驱动动画避免 React 重渲染（已有）

- **浏览器降级**：
  - `BrowserSpeechAdapter` 构造函数检测 API → unsupported 时 `VoiceControl` disabled+提示
  - 点击/文本入口完全独立于语音 API
  - E2E 验证："text input still works when voice may be unavailable"

## 6. 是否有遗留问题

- 已知问题：无阻断性问题。ESLint 有 4 个 warnings（预存）。`navigator.deviceMemory` 仅 Chrome 可用，其他浏览器回退到仅基于 `hardwareConcurrency` 的检测（预期行为）。
- 已接受风险：性能分级为启发式检测，`hardwareConcurrency` 不精确反映 GPU 能力。dpr 上限 1.5 和阴影降级作为安全网。后续可基于实际 FPS 测量动态调整 tier。
- 阻塞项：无。
