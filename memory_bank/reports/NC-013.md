## 3. 实现了哪些关键逻辑

- **性能分级检测**（`performanceConfig.ts`）：
  - 启发式检测：`navigator.hardwareConcurrency` ≥8 核或 `navigator.deviceMemory` ≥4GB → high；≥4 核 → medium；<4 核 → low；无法检测 → medium（安全默认）
  - 配置映射：high → dpr[1,1.5] + shadow 2048；medium → dpr[1,1.5] + shadow 1024；low → dpr[1,1.0] + shadow disabled
  - 降级顺序严格保证：像素比 → 阴影分辨率 → 阴影完全禁用；核心交互（点击、拖动、文本、语音）永不被降级
  - 结果缓存，`resetPerformanceConfig()` 用于测试重置

- **FPS 采样器**（`createFpsSampler`）：
  - 滑动窗口机制：收集 `delta` 样本，总和超过窗口（默认 1000ms）时裁剪最旧样本
  - 返回平滑 FPS（`samples.length / totalTime`）或 `null`（样本不足）
  - 忽略零/负 delta，支持 `reset()` 清空样本

- **VehicleScene 自适应渲染**：
  - `dpr` 从硬编码 `[1, 1.5]` 改为 `[1, perf.maxDpr]`
  - `SceneLights` 接收 `shadowMapSize` 和 `shadowsEnabled` props
  - `SceneGround` 条件渲染 `ContactShadows` 和 `receiveShadow`，low tier 时完全跳过阴影

- **响应式 CSS**：
  - `@media (max-height: 800px)`：控制面板 bottom/right 缩小、padding/gap 减半、状态项字号从 0.8→0.7rem、语音按钮从 44→36px、输入/按钮字号缩小
  - `@media (max-width: 1300px)`：控制面板 min/max-width 从 260/300→220/260px
  - 所有主要按钮命中区 ≥ 36×36px（满足 PRD 40×40px 近限）

- **浏览器降级验证**：
  - Web Speech API 检测：`BrowserSpeechAdapter` 构造函数自动检测 → unsupported 时 `VoiceControl` 显示降级文案 + 禁用按钮
  - 文本和点击入口完全独立于语音 API，任何浏览器均可使用
  - 已在 E2E 测试中验证（voice button renders, text input works when voice unavailable）

## 6. 是否有遗留问题

- 已知问题：无阻断性问题。ESLint 有 4 个 warnings（全部预存于 WindowInteraction.tsx）。`navigator.deviceMemory` 仅在 Chrome 中可用——其他浏览器回退到仅基于 `hardwareConcurrency` 的检测（此为预期行为）。
- 已接受风险：
  - **性能分级为启发式检测**：`hardwareConcurrency` 不精确反映 GPU 能力。在具有多核但弱 GPU 的设备上可能高估性能。`dpr` 上限 1.5 和阴影降级可作为安全网。后续可基于实际 FPS 测量动态调整。
  - **FPS 采样器未自动集成到生产代码**：`createFpsSampler` 为工具函数，需开发者在 `useFrame` 中手动调用。生产代码不主动监控 FPS 以避免开销。NC-014 可考虑是否在 dev 模式自动附加。
- 阻塞项：无。
